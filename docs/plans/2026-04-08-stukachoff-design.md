## Контекст

С 15 апреля 2026 года более 20 крупнейших российских платформ (VK, Яндекс, Сбер, Ozon, Авито, HH.ru, ВкусВилл и др.) обязаны по требованию Минцифры детектировать VPN-пользователей и передавать IP-адреса серверов в РКН. Угроза двухслойная: сетевой уровень (ТСПУ/DPI) и уровень приложений на устройстве.

Существующие инструменты (yourvpndead, RKNHardering) сами являются угрозой: отправляют IP VPN-сервера по HTTP, собирают fingerprint устройства, эксплуатируют локальные API без согласия пользователя.

---

## Что такое Stukachoff

Android-приложение которое показывает пользователю **кто на его устройстве "стучит" на его VPN** — и что с этим делать.

Не детектор VPN. Не скоринг. Верификатор защиты с конкретными actionable результатами.

**Позиционирование:** первый инструмент который реально на стороне пользователя.

---

## Философия и принципы

### Доказуемая безопасность
Core-функционал работает **только с `ACCESS_NETWORK_STATE`**. Без `INTERNET` permission физически невозможно отправить данные на внешний сервер — это техническая гарантия, а не маркетинговое заявление. Пользователь проверяет это в настройках Android.

### Два типа уязвимостей — разный UI
1. **Нескрываемые без root** — серый блок, объяснение без алармизма
2. **Устранимые** — светофор (🟢/🟡/🔴) с типом ущерба и инструкцией

### Честность важнее оптимизма
Факт VPN нельзя скрыть без root. Приложение говорит об этом прямо — и объясняет что именно можно защитить.

### YAGNI
Никаких аналитик, трекеров, фоновых сервисов, авто-обновлений. Только то что нужно для задачи.


## Три модуля

### Модуль 1: VERIFY

#### Предусловие
Перед сканированием — проверка статуса VPN:
- **VPN не обнаружен** → серый экран: *"Запусти VPN и повтори. Stukachoff показывает реальную картину только при активном соединении"*
- **Корпоративный VPN** (TYPE_VPN_PLATFORM) → уведомление об ограниченной применимости проверок

#### Блок А — "Это видят все" (серый, без светофора)

Показывается всегда. Не пугает — объясняет что именно знают приложения и чего НЕ знают.

| Что видно | Почему нельзя скрыть | Что знают / не знают |
|-----------|---------------------|----------------------|
| Факт VPN (`TRANSPORT_VPN`) | Ядро Android, `ConnectivityService` | ✅ VPN есть. ❌ Какой и куда |
| Имя интерфейса `tun0`/`wg0` | `java.net.NetworkInterface`, без разрешений | ✅ Тип клиента по имени |
| HTTP-пробы к заблокированным сайтам | Если VPN работает — сайты открываются | ✅ Косвенная детекция (VK MAX метод) |
| MCC + не-российский IP | Если у приложения `READ_PHONE_STATE` | ✅ Российская SIM + зарубежный IP |

Текст под блоком:
> *"Факт использования VPN скрыть без root невозможно — это архитектура Android. Stukachoff помогает защитить всё остальное."*

#### Блок Б — "Можно исправить" (светофор)

**1. Android-версия**
- 🟢 Android 10+ — `/proc/net/` закрыт через SELinux
- 🔴 Android 9 и ниже — любое приложение читает все активные TCP-соединения
- *Ущерб при 🔴:* IP всех серверов к которым подключён (включая SSH/VPS)

**2. Режим VPN-клиента**
- 🟢 TUN-режим — нет открытых прокси-портов на localhost
- 🟡 SOCKS5/HTTP на нестандартном порту — труднее найти
- 🔴 SOCKS5/HTTP на стандартном порту (1080, 10808, 10809, 7890, 7891, 2334, 2333)
- *Ущерб при 🔴:* протокол идентифицирован; при brute force — credentials

**3. xray gRPC API**
- 🟢 Порты 10085, 19085, 23456 закрыты
- 🔴 Открыт хотя бы один — без аутентификации
- *Ущерб при 🔴:* **критический** — IP сервера, UUID, SNI, ключи Reality → деанон + сервер в блоклист РКН

**4. Clash / Mihomo REST API**
- 🟢 Порты 9090/9091/19090 закрыты или `secret` установлен
- 🔴 Открыт без пароля
- *Ущерб при 🔴:* история соединений (какие сайты), IP серверов

**5. DNS**
- 🟢 Все запросы через туннель
- 🟡 DNS через туннель, но системный DNS провайдера активен параллельно
- 🔴 DNS-утечка — провайдер видит все запросы доменов
- *Ущерб при 🔴:* провайдер знает какие сайты посещаешь даже с VPN

**6. Системный прокси**
- 🟢 `http.proxyHost` и `socksProxyHost` не установлены (TUN-режим)
- 🔴 Установлены — все приложения видят адрес прокси без разрешений
- *Ущерб при 🔴:* IP прокси доступен через `System.getProperty()`

**7. MTU fingerprint**
- 🟢 MTU 1500 — стандартный
- 🟡 MTU 1420 — характерно для WireGuard
- 🔴 MTU нестандартный нижний — идентифицирует протокол
- *Ущерб при 🔴:* ТСПУ и приложения точнее идентифицируют тип VPN

**8. Exit IP** *(только full-версия, явный consent)*
- 🟢 Тестовый HTTPS-запрос ушёл через туннель, exit IP ≠ реальный IP
- 🔴 Запрос ушёл напрямую — VPN не перехватывает трафик
- Кнопка с объяснением: *"Делает один HTTPS-запрос. Данные не сохраняются."*

### Модуль 2: THREATS

#### Логика — двухуровневая

**Уровень 1 — разрешения** (не устаревает):
Анализ разрешений каждого установленного приложения → матрица возможностей.

**Уровень 2 — база известных** (обновляется через `threats.json`):
Подтверждённые и предполагаемые методы детекции конкретных приложений.

#### Матрица разрешений → возможности

| Разрешение | Что может детектировать |
|------------|------------------------|
| `ACCESS_NETWORK_STATE` | TRANSPORT_VPN, имя интерфейса, MTU |
| `INTERNET` | HTTP-пробы к заблокированным сайтам, GeoIP |
| `READ_PHONE_STATE` | MCC → корреляция с IP |
| `QUERY_ALL_PACKAGES` | Список установленных VPN-клиентов |
| `ACCESS_FINE_LOCATION` | WiFi BSSID → геолокация |
| *(нет разрешений)* | Сканирование localhost портов через `Socket` |

#### Три уровня угрозы

**🔴 Активный стукач** — подтверждённая детекция:
- VK Max: HTTP-пробы к заблокированным сайтам + PLMN + бинарный TCP-протокол (задокументировано, ntc.party тред 22584, март 2026)

**🟡 Потенциальный стукач** — разрешения есть, обязаны внедрить к 15.04.2026:
Сбер, Т-Банк, Ozon, Wildberries, Авито, Яндекс, Кинопоиск, HH.ru, ВкусВилл, МТС, Билайн, 2ГИС, Госуслуги

**⚪ Разрешений нет** — технически не может детектировать

#### UI карточки приложения

```
┌──────────────────────────────────────────┐
│  🔴  ВКонтакте / VK Max                  │
├──────────────────────────────────────────┤
│  Что знает прямо сейчас:                 │
│  • Факт VPN                              │
│  • Тип клиента (имя интерфейса)          │
│  • Доступность заблокированных сайтов    │
│  • Код оператора SIM                     │
│                                          │
│  Что НЕ знает при правильной настройке:  │
│  • IP твоего сервера                     │
│  • UUID / ключи шифрования               │
│                                          │
│  Ущерб: IP сервера → РКН → блоклист     │
│  [Как минимизировать →]                  │

```

#### Специальные блоки

**Операторы связи** — отдельный блок:
МТС, Билайн, МегаФон, Теле2 — детекция через ТСПУ (сетевой уровень), не через приложение. Знают MCC. Объяснение: другой уровень угрозы.

#### Обновление базы

```
Запуск THREATS
    ↓
Кэш < 7 дней? → использовать кэш
    ↓ нет (только full-версия)
Fetch threats.json через VPN-туннель
GitHub raw → SHA-256 верификация
    ↓
Смержить с локальной логикой разрешений
```

Если интернета нет — встроенный кэш из APK. Никогда не блокирует запуск.

### Модуль 3: LEARN

#### Принцип
Не отдельный экран — контекстный слой поверх VERIFY. Каждая красная/жёлтая карточка разворачивается в инструкцию на месте.

#### Структура контента для каждой проблемы

1. **Что это** — 2-3 предложения без жаргона
2. **Чем грозит** — конкретный тип ущерба
3. **Как исправить** — по клиентам: Hiddify / NekoBox / v2rayNG / Clash / Другой
4. **[Проверить снова]** — кнопка перезапуска конкретной проверки

#### Замкнутый цикл с ботом

Для пользователей VPN-сервиса с Telegram-ботом — кнопка в конце инструкций:

```
[Hiddify: инструкция]
[NekoBox: инструкция]
[v2rayNG: инструкция]
─────────────────────
Используете подписку?
[Обновить конфиг через бот →]
    ↓
t.me/BotName?start=stukachoff_fix_grpc
```

Deep link с контекстом проблемы → бот присылает исправленный конфиг.
Приватность: приложение не знает Telegram ID; бот не получает результаты сканирования.

#### Секция "Что нельзя скрыть"
Образовательный блок всегда доступный. Объясняет TRANSPORT_VPN на пальцах. Убирает паранойю — заменяет её пониманием.

#### Глоссарий
Любой технический термин в любом экране — кликабелен → bottom sheet с объяснением простым языком.

Термины: gRPC, TUN-режим, SOCKS5, DNS-утечка, MTU, TRANSPORT_VPN, UUID, SNI, Reality, split-tunnel, ASN, MCC, ТСПУ.

## Безопасность самого приложения

| Гарантия | Как проверить |
|----------|--------------|
| Нет INTERNET в core | Настройки → Приложения → Разрешения |
| Нет фоновых сервисов | Нет Service / WorkManager / JobScheduler |
| Нет аналитических SDK | apk analyzer — нет классов Firebase |
| Open source | GitHub — код открыт |
| Reproducible builds | Сборка из исходников = идентичный APK |

**Обновление APK:**
Никакого авто-обновления. При выходе новой версии — уведомление в LEARN. Пользователь скачивает вручную с GitHub, проверяет SHA-256.

## Открытые вопросы для реализации

1. **XHTTP**: изучить как транспорт для рекомендаций в LEARN (появился март 2026)
2. **Xiaomi / MIUI специфика**: MTU и интерфейсы могут отличаться — нужно тестирование
3. **GrapheneOS**: network permission toggle per-app меняет часть THREATS-логики
4. **Матрица клиент × проблема**: для каждого VPN-клиента точные инструкции как отключить gRPC, переключить на TUN — требует отдельного исследования
